# Plan: Unified Parameter Control Popup

## Goal
Menggabungkan Animation, CC Mapping, dan OSC Addressing ke dalam satu popup terpusat per parameter. Saat user klik ⚙, mereka bisa mengatur semua cara kontrol parameter tersebut.

## Arsitektur Saat Ini

```
Controls.jsx → AnimationPopup.jsx (hanya animation)
CcPanel.jsx → CC sliders + Input→CC mapping table
OscPanel.jsx → OSC address → CC mapping
App.jsx → ccMapping: { "Speed": "cc3" }
```

## Arsitektur Target

```
Controls.jsx → ParameterPopup.jsx (Animation + CC Source + OSC Address)
CcPanel.jsx → Hanya CC sliders (8 channel)
OscPanel.jsx → Hanya connection settings (host/port/connect)
App.jsx → parameterConfig: { "Speed": { cc: 3, oscAddr: "/fader1" } }
```

---

## Data Structure Baru

```javascript
// Sebelum (terpisah)
ccMapping = { "Speed": "cc3" }
// animasi di paramAnimation = { "Speed": { mode: 'sine', ... } }

// Sesudah (terpusat)
parameterConfig = {
  "Speed": {
    animation: { mode: 'sine', speed: 1, min: 0, max: 1, ... },
    cc: 3,           // CC channel (1-8) atau null
    oscAddr: "/fader1" // OSC address atau null
  }
}
```

## Implementation Steps

### Step 1: Update App.jsx State
```javascript
// Ganti ccMapping dengan parameterConfig
const [parameterConfig, setParameterConfig] = useState({})

// Handler baru
const handleParameterConfigChange = useCallback((paramName, field, value) => {
  setParameterConfig(prev => ({
    ...prev,
    [paramName]: { ...prev[paramName], [field]: value }
  }))
}, [])
```
